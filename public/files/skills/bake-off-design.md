# Bake-Off Design — Trial & Evaluation Framework

## When to use
Use when designing a free trial, product comparison, vendor evaluation, or head-to-head test that prospects can run to compare your product against competitors. Works for API products, SaaS tools, data providers, or any product where performance can be measured objectively.

## Core Principle

A well-designed bake-off is your best sales tool when your product is genuinely better. Structure the evaluation so the most important dimensions are the ones where you win. Stay fair enough that the framework is credible. The prospect should feel like they discovered your advantage, not that you tricked them into it.

## Framework Design

### Step 1: Define Evaluation Dimensions

List 6-10 dimensions that matter for the product category. Include:
- Dimensions where you clearly win (your main displacement angles)
- Dimensions where the competitor wins (builds credibility)
- Dimensions that are table stakes (both pass)

**Rules:**
- Every dimension must be measurable. No subjective criteria like "user experience" unless you define a specific test.
- Each dimension gets a weight (percentage of total score, summing to 100%).
- Your strongest dimensions should carry the highest weights. This is where the structural advantage lives.
- Don't make it obvious. The weighting should feel reasonable to anyone reading it.

**Example weighting strategy:**
- Your top 2 advantages: 15-20% weight each (30-40% total)
- Your secondary advantages: 10-15% weight each
- Competitor's advantages: 5-10% weight each
- Table stakes: 5-10% weight each

### Step 2: Design the Test Protocol

For each dimension, specify:
1. **What to test** — Exact steps the prospect takes
2. **How to measure** — Quantitative metric (%, count, yes/no)
3. **What good looks like** — Benchmark or reference point
4. **How to score** — 1-5 scale with clear definitions

**Scoring rubric (standard):**
| Score | Meaning |
|-------|---------|
| 5 | Best-in-class. Exceeds requirements. |
| 4 | Strong. Meets all requirements. |
| 3 | Adequate. Meets basic requirements. |
| 2 | Below average. Gaps in coverage. |
| 1 | Poor or missing. Does not meet requirements. |

### Step 3: Build the Scoring Spreadsheet

Create a simple spreadsheet with:
- Rows: Each evaluation dimension
- Columns: Weight, Provider A (your product) score, Provider A weighted, Provider B score, Provider B weighted
- Auto-calculated total row
- Pre-fill the weights. Leave scores blank for the prospect to fill.

### Step 4: Write the Evaluation Guide

The guide should include:
1. **Introduction** — What this framework evaluates and why these dimensions matter
2. **Dimension descriptions** — What each dimension tests, why it matters, what good looks like
3. **Step-by-step test instructions** — Exact protocol for running the bake-off (with time estimate)
4. **How to interpret results** — What score gaps mean and when to switch
5. **FAQ** — Common questions about the evaluation process

### Step 5: Package as Lead Magnet

The bake-off kit becomes a lead magnet:
- Evaluation guide (PDF or web page)
- Scoring spreadsheet (downloadable CSV/XLSX)
- Free trial credits to run the test
- Contact for questions

Gate it behind an email capture. The person downloading this is a high-intent lead.

## Design Principles

### Be honest about competitor strengths
If your competitor wins on a dimension, say so in the guide. "Provider X has strong real-time company signals. If that's your primary need, they're a reasonable choice for that use case." This builds trust and makes your advantages feel more credible.

### Make the test easy to run
Total time should be under 30 minutes. If the bake-off is too complex, prospects won't finish it. Break it into clear steps with time estimates for each.

### Use the prospect's own data
Synthetic data tests are unconvincing. The bake-off should use the prospect's actual CRM records, customer file, or production data. This makes the results personally relevant and harder to dismiss.

### Structure for your advantage without being manipulative
- Weight your strongest dimensions highest, but keep weights defensible
- Include dimensions where the competitor wins (at lower weights)
- The total score should reflect genuine product-market fit, not gaming

### Include a "run it for them" option
Many prospects won't run the bake-off themselves. Offer to run it for them: "Send us [X records] and we'll return results in [timeframe]." This removes friction and gets you their data, which is the most valuable thing.

## Common Bake-Off Dimensions (by category)

### Data/API Products
- Field depth (total fields returned)
- Match rate by identifier type
- Data quality (fill rates, accuracy)
- Data freshness / update frequency
- Data standardization
- Consumer vs professional coverage
- Identity resolution capability
- Compliance and certifications
- API design and documentation
- Pricing model and unit economics

### SaaS Tools
- Feature completeness for core use case
- Time to value (setup to first result)
- Integration depth with existing stack
- Performance / speed
- Reporting and analytics
- Support responsiveness
- Pricing transparency
- Data portability
- Security and compliance
- Roadmap alignment

### Results Interpretation Guide

Include guidance on what score gaps mean:
- **0-5% difference:** Essentially tied. Decide on other factors (pricing, support, roadmap).
- **5-15% difference:** Meaningful advantage. Worth switching if the winning dimensions match your primary use case.
- **15%+ difference:** Clear winner. The gap will compound over time as you build on the data/product.

## Anti-Patterns

1. **Don't make it too long.** 8-10 dimensions max. More than that and prospects won't finish.
2. **Don't weight everything equally.** Equal weights = no structural advantage. Weight by what matters most for the use case.
3. **Don't skip the competitor's strengths.** An evaluation framework that your competitor would call unfair loses credibility.
4. **Don't require technical setup.** The bake-off should work with a CSV upload or simple API call. No SDK installation.
5. **Don't hide behind the framework.** If a prospect runs the bake-off and the competitor wins, be honest about it. "If [competitor] wins on the dimensions that matter most to you, stick with them." This builds long-term trust.
